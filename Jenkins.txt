			                                       JENKINS
This is a sample Python application that was dockerized and validated through Jenkins pipeline


Jenkins controller coordinates the job, but the agent performs the work. So the tools required by the pipeline must exist on that agent.
the Jenkins agent (slave) is the machine that actually executes the pipeline.
The Jenkins agent itself needs Java because Jenkins agents run using a Java process


File contents on GIT Bash:

app.py
===========

from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello():
    return "Hello from Jenkins pipeline + Docker"

if __name__ == '__main__':
   app.run(host='0.0.0.0' , port=5000)

requirements.txt
================

flask

Dockerfile
==========

FROM python:3.11-slim

WORKDIR /app

COPY . .

RUN pip install -r requirements.txt

EXPOSE 5000

CMD ["python", "app.py"]

Jenkinsfile
===========

pipeline {
    agent {
        label 'myslave'
    }

    stages {

        stage('Checkout Code') {
            steps {
                git branch: 'main',
                    url: 'https://github.com/awsarchitect2030/Jenkins_python_pipeline.git'
            }
        }

        stage('Build Docker Image') {
            steps {
                sh 'docker build -t python-flask-demo .'
            }
        }

        stage('Remove Old Container') {
            steps {
                sh 'docker rm -f flask-container || true'
            }
        }

        stage('Run Docker Container') {
            steps {
                sh 'docker run -d -p 5000:5000 --name flask-container python-flask-demo'
            }
        }
    }
}

Launch EC2 instance  - Below commands can be used on Linux machine.

1. Install java

sudo yum install java-21-amazon-corretto -y

java -version

2. Install GIT

sudo yum install git -y

3. Install Jenkins : 

sudo wget -O /etc/yum.repos.d/jenkins.repo \
https://pkg.jenkins.io/redhat-stable/jenkins.repo

sudo rpm --import https://pkg.jenkins.io/redhat-stable/jenkins.io-2023.key

sudo yum install jenkins -y

sudo systemctl start jenkins
sudo systemctl enable jenkins
sudo systemctl status jenkins

If status becomes active, get the admin password:

sudo cat /var/lib/jenkins/secrets/initialAdminPassword

4. Then open:

http://<EC2-public-IP>:8080 (Make sure you enable port 8080 - Custom TCP before launching) 
Use the admin password from above step and set up your credentials.

5. Install Maven on EC2

sudo yum install maven -y

mvn -version


6. New job :

Git  hub : https://github.com/awsarchitect2030/jenkins-devops.git


7. Create 2nd EC2 instance as slave and create passwordless authentication. 

8. On the first (Master instance)

ip addr show - to know IP address

Follow below steps to generate keys.
sudo su - jenkins
id jenkins
grep jenkins /etc/passwd
sudo -u jenkins ssh-keygen -t rsa -b 4096
sudo -u jenkins cat /var/lib/jenkins/.ssh/id_rsa.pub (Copy the contents and paste under /home/ec2-user/.ssh/authorized_keys on slave machine)
sudo -u jenkins ssh ec2-user@172.31.44.23 (Should connect without prompting password)

10. On slave machine


ip addr show
vi /home/ec2-user/.ssh/authorized_keys
sudo yum install java-21-amazon-corretto -y
java -version

Below steps are to set path on Jenkins pipeline to run the job.
mkdir -p /home/ec2-user/Jenkins
chmod 755 /home/ec2-user/Jenkins

ec2-user need to have permission to access the Docker socket so execute below.

sudo usermod -aG docker ec2-user
newgrp docker

docker ps
docker images
sudo yum install git -y
groups ec2-user
id

===============================================================

Jenkins Job set up

Open Jenkins page 

Lets set up slave node on Jenkins.

Manage Jenkins -> Nodes -> New node -> Permanent Agent -> 
Remote root directory (/home/ec2-user/Jenkins) 
Labels (myslave)
Launch method -> Launch agents via ssh -> Host -> Enter Slave instance IP -> credentials -> Add -> SSH username with private key ->ec2-user as username and for Private Key - Enter directly paste value from master instance /var/lib/jenkins/.ssh/id_rsa file (private key content of generated key)
Host Key Verification Strategy ->  Known hosts file Verification Strategy.

Goto Homepage and enter item name , select pipeline -> Enter description -> Pipeline -> Definition -> Pipeline script from SCM -> SCM -> GIT -> Enter full GitHub URL for Repository URL and Credentials can be left untouched for public repositories.

Branches to build -> Branch Specifier -> Update the value from */master to */main , Script Path -> Jenkinsfile 

Save the job and run using "Build Now". Check on Console output of ran build and troubleshoot as per failures. 

Fix and rerun the build.
