1. Create an ubuntu instance on AWS console.

2. Connect to the instance and run "sudo apt update"

3. Run "sudo apt install docker.io -y"

4. To pull any image from Docker hub
sudo docker pull ubuntu

5. To check if images is downloaded
sudo docker images

sudo docker --version

6. To create a container (interactive terminal access and background) , run the image.
sudo docker run -itd -p 80:80 ubuntu 

7. To check running containers : sudo docker ps  (sudo docker ps -a will return stopped container as well)

8. To login to ubuntu container 
sudo docker exec -it 1cfb5376b49a /bin/bash

9. Once logged in, run "apt update"

10. To install apache2 inside the container, run "apt install apace2 -y"
To check status : service apache2 status
To start apache2 : service apache2 status

root@1cfb5376b49a:/# service apache2 status
 * apache2 is not running
root@1cfb5376b49a:/# service apache2 start
 * Starting Apache httpd web server apache2                                                                                                 AH00558: apache2: Could not reliably determine the server's fully qualified domain name, using 172.17.0.2. Set the 'ServerName' directive globally to suppress this message
 * 
root@1cfb5376b49a:/# service apache2 status
 * apache2 is running
root@1cfb5376b49a:/# 

Next allow http traffic on inbound security group manually on the instance so we can validate the traffic from browser.

Once done copy the public IP and test http://public IP:80 and it should display Apache2 home page.
